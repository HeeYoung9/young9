통신프로토콜(http/https)

컴퓨터사이에서 정보(데이터)를 주고 받을때 하는 약속
1) 어떤 방식으로 접속
2) 어떤 속도로 전송
3) 어떤 형태로 전송

TCP/IP

통신 프로토콜 중 하나. [com1] [com2]
애플리케이션층 <-가상 데이터 송수신-> 애플리케이션층
트랜스포트층(TCP/UDP) 트랜스포트층(TCP/UDP)
네트워크층(인터넷층 IP) 네트워크층(인터넷층 IP)
데이터링크층 <-실제 데이터 송수신-> 데이터링크층

애플리케이션층: 실제 인터넷 응용 프로그램 실행
트랜스포트층(TCP/UDP): 데이터 전송지확인, 데이터 에러체크
네트워크층(인터넷층IP): 데이터를 패킷단위로 변환 정보 전달
데이터링크층: 랜카드 제어, H/W제어

서버&클라이언트

서버: 정보/서비스를 제공하는 컴퓨터 (성능이 월등히 좋음)
클라이언트: 정보/서비스를 요청하고, 제공받는 컴퓨터

IP주소

ex) 192.168.7.200 (4자리 0~255)
인터넷상에서 특정 컴퓨터를 구분하기 위한 값 (예. 전화번호)
-localhost: 내컴퓨터(127.0.0.1)
->www.naver.com (도메인주소: IP주소를 대신하는 이름)
DNS서버: IP주소<->도메인주소 변환 서버

포트(port) 번호

컴퓨터에는 인터넷을 사용하는 프로그램이 다양함
프로그램을 구분하는 고유번호 (같은번호를 쓰면 충돌 발생)
일반적으로 1024아래 번호는 사용x
tomcat 포트번호: 8080->8088 (오라클이랑 충돌발생할까봐 미리 바꿨음)
mysql 포트번호: 3306
oracl 포트번호: 8080

(이미지, html, css, js,...) 클라이언트 ->요청(naver.com)-> 서버(WEB):정적데이터만 처리 ->서버(WAS): 동적데이터 처리 ->데이터베이스(mysql) ㅣ요청준비(html파일) ㅣ 클라이언트 <-응답 <-서버(WEB) <-서버(WAS) <- 웹브라우저실행 우리는 웹서버와 was서버를 통합해서 tomcat 이용
